ul은 unordered list 의 약자이다.
사전적 의미는 중요하지 않다.
속성값은 3개, circle, disc, square 가 있으나, 이것 역시 속성값을 외울 필요는 없다. html5에서는 지원 하지 않기때문.
중요한 것은 레이아웃을 구성 할때, navigation, board, sitemap 등 목록 형태의 컨텐츠를 표현하는 중요한 레이아웃 블럭 이라는 점만 기억 하면 되겠다.

ol은 ordered list의 약자이다.
사전적 의미는 중요하지 않다.
속성값은 4개(+@) "A, a, I, i"가 있다.(css처리)
중요한 것은 layout 구성 시, 번호로 정렬 되어야 하는 (순차적)목록 형태의 content를 표현하는 block(display: list-item / ul도 동일)이라는 점.

HTML
Hyper Text Markup Language

dl 은 definition list의 약자.
정의형 목록인데, 마찬가지로 사전적 의미는 비중요.
특별한 속성값은 없다.
한개의 title(dt)을 기준으로 뒷받침 하는 내용의 data(dd)를 추가 삽입할 때 유리.
dt와 dd의 갯수가 딱히 정해지지는 않으나 dt x 1 dd x 여러개를 추천.
